# Задание 4. Пространственная привязка
Работа осуществляется в редакторе Python консоли QGIS. 
-	Импортировав gdal, открыть растровое изображение без привязки из папки no_crs (номер изображения указан пофамильно в таблице Задание_4) 
-	Для каждого канала от 1 до RasterCount+1 (тут отсчёт ведётся не с нуля), получить этот канал (GetRasterBand), затем преобразовать каждый из них в массив (ReadAsArray), где тип данных (astype) – с плавающей запятой. Каждый из этих массивов добавить в заранее созданный пустой список массивов
- Создать копию исходного изображения (см. задание 3, часть с созданием растрового изображения, за исключением того, что для индексного изображения подавался один массив (канал), а сейчас – несколько (т.к. это RGB), т.е. для каждого массива из созданного ранее списка необходимо создать свой канал (GetRasterBand и WriteArray))
-	Далее необходимо получить 4 угловые координаты границ, хранящиеся в файле footprint_n, где n – номер Вашего изображения. Один из возможных вариантов получения: 
<br>•	Любым из способов (по имени, активный, …) открыть векторный слой footprint_n
<br>•	Получить нулевой (т.к. в нашем случае он единственный) объект (getFeature)
<br>•	Для него получить геометрию (geometry), которую считать, как полигон (asPolygon)
<br>•	Для каждой из 4 точек полигона (представленных, как QgsPointXY) задать кортеж: точка.x(), точка.y()
<br>•	Поместить каждый из таких кортежей в заранее созданный пустой список
-	Созданной ранее копии растрового изображения задать проекцию (SetProjection), как у векторного слоя (crs и authid)
-	Создать список, в которого войдут 4 gdal.GCP(x, y, z, pixel, line). При этом x и y берутся из списка с кортежами точек (список[номер_точки][номер_координаты]), z в данном случае всегда 0, pixel – 0 или RasterXSize-1, line – 0 или RasterYSize-1. При этом первая точка (список[0][0], список[0][1])– северо-западная, далее – по часовой стрелке
-	Установить опорные точки для скопированного ранее растрового изображения (SetGCPs), подав в качестве аргумента список GCP и проекцию растрового изображения (GetProjection), после чего очистить кэш (изрбражение.FlushCache())
