# Програмирование картографо-геодезических задач
<p>Каждая ветка данного репозитория представляет собой практическое задание.</p>
<p>
Процесс работы может строится следующим образом:
<li>Зарегистрировавшись на GitHub, создать личную попию данного репозитория
    <br>&emsp;&nbsp;Для этого необходимо нажать <i>Fork</i> и убрать галочку <i>Copy the intro branch only</i></li>
<li>Установить <a href="https://git-scm.com/downloads">Git</a></li>
<li>Открыв Git Bash, сменить директорию на желаемую с помощью команды <code>cd <i>path</i></code>
    <br>&emsp;&nbsp;(далее сюда будет добавлена папка с репозиторием)</li>
<li>Зайдя в своём профиле в личную копию репозитория, скопировать его HTTPS, нажав на <i>Code</i></li>
<li>В Git Bash клонировать данный репозиторий в выбранную директорию: <code>git clone <i>https</i></code></li>
<li>Вновь сменить директорию с помощью <code>cd gis-programming-class</code>, получив доступ к репозиторию</li>
<li>Выбрать соответствующую ветку с помощью <code>git fetch origin <i>assignment_n</i></code>, где <i>n</i> – номер задания</li>
<li>Переключиться на неё с помощью команды <code>git switch <i>assignment_n</i></code></li>
<li>Унаследоваться от этой ветки с нужным заданием: <code>git checkout -b <i>n_lastname</i></code></li>
<li>В проводнике переместить файлы, являющиеся ответом на задание, в выбранную директорию
    <br>&emsp;&nbsp;– Если ответ – ссылка, оставить в ветке только README.md, заменив его текст на свою ссылку. Остальное – удалить
    <br>&emsp;&nbsp;– Если ответ – файл, очистить директорию от файлов задания и добавить только файл ответа</li>
<li>В Git Bash добавить эти файлы с помощью команды <code>git add .</code></li>
<li>Убедиться, что файлы добавлены можно с помощью <code>git status</code></li>
&emsp;&nbsp;Добавленные файлы отразятся зелёным, недобавленные – красным
<li>Внести изменения с помощью <code>git commit -m "<i>n Фамилия</i>"</code></li>
<li>Отправить изменения из локального репозитория в удалённый: <code>git push origin <i>n_lastname</i></code></li>
<li>В GitHub создать запрос на слияние (pull request): 
    <br>&emsp;&nbsp;– Из ветки <i>n_lastname</i> Вашей копии данного репозитория
    <br>&emsp;&nbsp;– В ветку <i>assignment_n</i> оригинального репозитория</li>
<li>Баллы за задание или необходимость доработки будут указаны в комметариях к запросу после проверки</li>
</p>
