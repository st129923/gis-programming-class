# Задание 6. Создание программного модуля QGIS
В рамках данного задания необходимо создать плагин для QGIS, который бы реализовывал поставленную Вами простую задачу (выполнение любых операций с векторными или растровыми слоями: изменение геометрии или атрибутов, перепроецирование, изменение пространственного разрешения…).
<br>Важно, чтобы задача у каждого была своя. В противном случае, оценка выставляется только первому сдавшему, остальным – 0 баллов.
<br>Наиболее упрощенный вариант выполнения работы – использование модуля Plugin Builder:
- Установив (Plugins – Manage and Install Plugins) и открыв Plugin Builder, заполнить необходимые поля, расписав, что должен делать разработанный Вами модуль. В качестве названия модуля указать свою фамилию. Путь для сохранения на Windows:
<br><code>C:/Users/*имя_пользователя*/AppData/Roaming/QGIS/QGIS3/profiles/default/python/plugins</code>
<br>![plugin_builder](https://github.com/odinkomnogim/gis-programming-class/blob/assignment_6/illustrations/plugin_builder.png?raw=true)
- Перезагрузив QGIS, необходимо включить модуль (поставить галочку) в меню плагинов и убедиться, что шаблон плагина появился на панели инструментов
<br>![enable_plugin](https://github.com/odinkomnogim/gis-programming-class/blob/assignment_6/illustrations/enable_plugin.png?raw=true)
- При нажатии на иконку плагина должно открываться пустое диалоговое окно. Чтобы это изменить, необходимо воспользоваться Qt Designer, который, вероятно, будет находиться в папке bin в директории, куда установлен QGIS. Найти можно, кликнув на ярлык QGIS – расположение файла – qgis-designer.bat (возможно, потребуется запуск от имени администратора)
- Открыв в Qt Designer файл Вашего модуля с расширением .ui, правой кнопкой мыши задать Lay out – Lay Out in a Grid
<br>![designer_layout](https://github.com/odinkomnogim/gis-programming-class/blob/assignment_6/illustrations/designer_layout.png?raw=true)
- Далее следует найти необходимые для реализации Вашей задумки элементы в левой панели и перетащить их в область диалогового окна. Это может быть что угодно: виджет выбора файлов (QgsFileWidget), копка (QPushButton), поле со списком (QComboBox) и т. д.
<br>![designer_widgets](https://github.com/odinkomnogim/gis-programming-class/blob/assignment_6/illustrations/designer_widgets.png?raw=true)
- Сохранив .ui файл, необходимо проверить его в QGIS. Перезагрузка разрабатываемого модуля требуется при тестировании плагина после любого внесения изменений. В противном случае, эти изменения не будут отражаться. Рекомендуется установить модуль Plugin Reloader (однако это не обязательно: перезагрузку плагина в процессе разработки можно осуществлять путём снятия и повторной установки галочки в списке плагинов)
<br>![plugin_reloader](https://github.com/odinkomnogim/gis-programming-class/blob/assignment_6/illustrations/plugin_reloader.png?raw=true)
- Когда готов интерфейс, необходимо подключить его к выполняемым функциям. Для работы с виджетами существует различное количество сигналов (например, сигнал clicked для QPushButton), позволяющих связать нажатие на кнопку или другое изменение состояния с какой-либо функцией. Однако в данном простом примере в шаблоне уже добавлен QDialogButtonBox, где при нажатии на OK происходит выполнение описанных функций
- Останется только дописать код после if result в функции run главного файла плагина. Для примера: буферизация выбранного в виджете файла и добавление временного слоя с буферными зонами на карту (буферизацию, как и другие возможности, которые были непосредственно рассмотрены в рамках прошлых практических, прошу не реализовывать, либо реализовывать в совокупности с чем-то ещё для выполнения более сложной задачи)
<br>![script_section](https://github.com/odinkomnogim/gis-programming-class/blob/assignment_6/illustrations/script_section.png?raw=true)
- Убедившись, что модуль выполняет поставленную Вами задачу, перенести файлы плагина в ветку 6_Фамилия Вашей копии данного репозитория и, как и в других заданиях, создать Pull request в ветку assignment_6
<br>![pushed_branch](https://github.com/odinkomnogim/gis-programming-class/blob/assignment_6/illustrations/pushed_branch.png?raw=true)
